Overview

The Dual Motor Driver for ESP32 is a custom-designed PCB that provides a complete motor control solution for robotics and automation projects. Built specifically for the popular ESP32 WROOM-DA module, this compact board offers professional-grade motor control in a maker-friendly package.

Dual Motor Driver PCB - Top View

Key Features

Custom PCB Design

Professionally designed PCB optimized for motor control

Wide Voltage Range

Supports input voltage from 7V to 25V, making it compatible with various battery configurations and power supplies.

PCB component layout and pin assignments

Onboard Components

  • Motor Driver ICs: High-efficiency H-bridge drivers for dual motor control
  • LED Indicators: Status LEDs for motors, sensors, and power
  • Test Button: Hardware button for motor functionality testing
  • Buzzer: Audio feedback for system alerts and notifications
  • Selection Switch: Multi-position switch for mode selection
  • Sensor Inputs: Multiple GPIO pins for various sensor types
  • Servo Headers: Dedicated pins for servo motor control
  • Communication Headers: UART, I2C, and PWM breakout pins

Getting Started

Basic Setup

  1. Insert ESP32 WROOM-DA module into the socket
  2. Connect power supply (7-25V) observing polarity
  3. Connect motors to the motor output terminals
  4. Upload your code via USB or wireless programming

Example Code - Buzzer Control

Here's a simple example to test the onboard buzzer functionality:

#include 

// Pin definitions for Dual Motor Driver
#define BUZZER_PIN 23
#define LED_MOTOR1 18
#define LED_MOTOR2 19
#define TEST_BUTTON 21

void setup() {
  Serial.begin(115200);
  
  // Initialize pins
  pinMode(BUZZER_PIN, OUTPUT);
  pinMode(LED_MOTOR1, OUTPUT);
  pinMode(LED_MOTOR2, OUTPUT);
  pinMode(TEST_BUTTON, INPUT_PULLUP);
  
  // Startup sequence
  Serial.println("Dual Motor Driver - Buzzer Test");
  startupTone();
}

void loop() {
  // Check test button
  if (digitalRead(TEST_BUTTON) == LOW) {
    Serial.println("Test button pressed!");
    testSequence();
    delay(500); // Debounce
  }
  
  delay(100);
}

void startupTone() {
  // Play startup melody
  int melody[] = {262, 294, 330, 349}; // C, D, E, F
  int duration = 200;
  
  for (int i = 0; i < 4; i++) {
    tone(BUZZER_PIN, melody[i], duration);
    digitalWrite(LED_MOTOR1, HIGH);
    delay(duration);
    digitalWrite(LED_MOTOR1, LOW);
    delay(50);
  }
  
  noTone(BUZZER_PIN);
}

void testSequence() {
  // Audio and visual feedback
  for (int i = 0; i < 3; i++) {
    tone(BUZZER_PIN, 1000, 100);
    digitalWrite(LED_MOTOR1, HIGH);
    digitalWrite(LED_MOTOR2, HIGH);
    delay(100);
    
    noTone(BUZZER_PIN);
    digitalWrite(LED_MOTOR1, LOW);
    digitalWrite(LED_MOTOR2, LOW);
    delay(100);
  }
}

// Additional functions for motor control
void motorForward(int speed) {
  // Motor control implementation
  analogWrite(MOTOR1_PWM, speed);
  analogWrite(MOTOR2_PWM, speed);
  // Set direction pins
}

void motorStop() {
  analogWrite(MOTOR1_PWM, 0);
  analogWrite(MOTOR2_PWM, 0);
}

Advanced Features

Sensor Integration

The board provides multiple input pins for various sensors commonly used in robotics:

  • Ultrasonic distance sensors (HC-SR04)
  • Line following sensors
  • Encoders for motor feedback
  • Limit switches and bumper sensors
  • Analog sensors (potentiometers, light sensors)

Communication Interfaces

Breakout headers provide access to:

  • UART: For serial communication with external devices
  • I2C: For sensor networks and display modules
  • PWM: For servo control and custom motor drivers
  • WiFi/Bluetooth: Wireless control and monitoring via ESP32
